Mariposa

use your voice to move the butterfly!

p5.js * HTML * CSS

Context

For the first project of the Ironhack web development bootcamp we had 3 days to create a game in p5.js. I created a “run and jump” game that responds to sound. The goal of the game is to catch as many light orbs as possible in the given time. The butterfly responds to frequency.

Development

I used the p5.sound library, using .getEnergy() to obtain the frequency input given by the player. The designs for the backgrounds, I found on itch.io. As you might be able to imagine, debugging was a fun process, as I had to sing and scream to my screen.

During this projectweek I won the price for “the most creative game”

Gerrit

chat with random people, near to you!

React * MongoDB * Mongoose * Express * socket.io * Node.js * CSS * HTML

go to web application

see code

Context

For the last project of the Ironhack bootcamp we had to create a full-stack MERN (MongoDB, Express, React, Node.js) web application. In teams of 3, we had one week to make it happen. We decided to create a mobile web application that allows users to chat with others only while being in the same location. The idea behind it was to create an environment where neighbors could talk to each other (for example if someone lost their cat/keys), tourists could ask for restaurant tips while being in a specific area and people in- or outside a club can communicate with each other concerning the queue, djs and vibes. Users create a profile but can still stay anonymous, giving them a bit more freedom to express themselves. No one can enter the chat box while not being in the specific location it concerns.

Development

This was my first project in React and I can say it surely made the implication of the front end a bit easier. It was a very pleasant experience. The chat was implemented using socket.io. Working in a team of 3 taught me to communicate clearly.

Tibetan Singing Robot

forget about your phone, meditate and feel the vibrations

Arduino (C++)

see code

Context

I am a daily meditator. I usually use a youtube-video that plays a tone when the time is up. For me, this became a problem since I always had to have a look at my phone before starting. Instead of being influenced by messages or the temptation of social media I prefer to start my meditation with a clean mind. I optimized the situation by creating a physical meditation timer, combining a Tibetan singing bowl and Arduino.

Development

Using Arduino hardware, I soldered a switch, display and motor to an Arduino UNO. Users can set the amount of minutes they want to meditate, click to start and than have 5 seconds to sit. The motor plays the bowl at the start and at the end of the meditation.

Context

Music visualization

alter your songs and learn about the mechanism behind them

JavaScript * d3.js * HTML * CSS

go to website

see code

Context

For the final project of my Computer Science minor we had to create a data visualization linking 3 different graphs using d3.js. I chose to use music as my data and also added a “synthesizer” to give the user the possibility to manipulate sound and graphs, creating a learning environment. During that time I didn’t have enough time to work on the styling so I decided to give my old project new life. I fixed some bugs, changed the styling and made sure that users can upload their own songs.

Development

The AudioContext WebAPI, helped me to retrieve and manipulate data from different songs. I obtained the frequency and wavelength to form a bar chart and circle chart, using recursive functions to create live-updating charts.
